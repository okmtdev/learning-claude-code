# 03. 主要機能（編集・検索・Git・計画モード）

ここからは `sandbox/` を題材に、実際にコードを触ります。
各セクションは Claude Code のプロンプトに **そのままコピペ** して試せます。

> 安全のため、各タスクの前に `git status` がクリーンであることを確認しておくと、
> 変更点を `git diff` で見比べやすくなります。

## 1. コードを検索・理解する

```
sandbox 配下で「平均」を計算している関数はどこ?ファイルと行番号で教えて。
```

```
slugify 関数は何をする関数? 入出力の例を3つ挙げて。
```

Claude は Grep / Glob / Read を使ってコードを探し、`file:line` 付きで答えます。

## 2. コードを編集する

### 機能追加（テスト駆動）

```
sandbox/src/textUtils.js に capitalize(text) を追加して。
先頭文字を大文字にし、残りはそのまま返す。
あわせて sandbox/test/textUtils.test.js にテストも追加して、npm test が通ることを確認して。
```

確認ポイント:
- Claude が編集の差分（diff）を提示し、適用してよいか確認してくる。
- 適用後に `cd sandbox && npm test` を実行し、テストが通ることを見せてくれる。

実行後、自分でも差分を確認:

```bash
git diff
cd sandbox && npm test
```

### バグ修正の練習

わざとバグを仕込んで、直してもらう流れも試せます:

```
sandbox/src/mathUtils.js の average 関数を、わざと「合計を要素数+1で割る」バグに書き換えて。
その後、npm test を実行してテストが失敗することを確認して。
```

失敗を確認したら:

```
さっきのバグを修正して、テストが通る状態に戻して。
```

## 3. 計画モード（Plan Mode）を使う

大きめの変更は、いきなり編集させず「計画 → 承認 → 実行」が安全です。

1. `claude` 起動中に `Shift+Tab` を押して **Plan Mode** にする。
2. 依頼する:

```
sandbox に「配列を size ごとに分割する chunk(array, size) 関数」を追加したい。
src への追加・テスト追加・実行までの計画を立てて。まだ変更はしないで。
```

3. Claude が計画を提示する（この間ファイルは変更されない）。
4. 内容に納得したら計画を承認 → 実行に移る。

## 4. Git 操作を任せる

Claude Code は Git も扱えます。

```
いまの変更内容を git diff で確認して、内容にふさわしいコミットメッセージで commit して。
```

> プッシュなど外部に影響する操作は、明示的に頼んだときだけ実行されます。

## 5. CLAUDE.md でプロジェクトの文脈を記憶させる

`CLAUDE.md` はプロジェクトのルートに置く「Claude への常駐メモ」です。
コーディング規約・よく使うコマンド・ディレクトリの意味などを書いておくと、毎回伝えなくて済みます。

自動生成を試す:

```
/init
```

または `#` で追記:

```
#このリポジトリのテストは sandbox ディレクトリで `npm test` を実行する
```
