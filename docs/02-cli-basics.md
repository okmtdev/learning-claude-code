# 02. CLI の基本操作

Claude Code を「対話的に使う」基本を押さえます。コピペで順に試してください。

## 起動モード

| コマンド | 説明 |
|---|---|
| `claude` | 対話モードで起動（基本） |
| `claude "プロンプト"` | 起動と同時に1つ依頼 |
| `claude -p "プロンプト"` | **非対話（print）モード**。結果だけ出力して終了。スクリプト向き |
| `claude -c` | 直前のセッションを継続（continue） |
| `claude -r` | 過去セッションを選んで再開（resume） |

### 試す: 非対話モード

ターミナル（Claude のプロンプトではなく通常のシェル）で:

```bash
claude -p "sandbox/src/mathUtils.js にある関数名だけを箇条書きで挙げて"
```

パイプにも繋げられます:

```bash
cat sandbox/src/textUtils.js | claude -p "この中で一番複雑な関数はどれ?理由も1行で"
```

## 対話中の操作

`claude` を起動した状態で使える基本操作:

| 入力 | 効果 |
|---|---|
| 普通の文章 | 依頼を送る |
| `/` から始まる入力 | スラッシュコマンド（[docs/04](04-slash-commands.md)） |
| `@ファイル名` | ファイルを文脈に添付（補完が効く） |
| `#メモ` | メモを `CLAUDE.md` などに記憶させる |
| `Esc` | 生成を中断 |
| `Esc` 2回 | 履歴を遡って編集 |
| `Shift+Tab` | 権限モードの切替（後述） |
| `Ctrl+C` / `/exit` | 終了 |

### 試す: ファイル添付

```
@sandbox/src/textUtils.js この truncate 関数の境界値の挙動を説明して
```

## 権限モード（重要）

Claude Code は、ファイル編集やコマンド実行の前に許可を求めます。`Shift+Tab` でモードを切り替えられます:

| モード | 挙動 |
|---|---|
| 通常（default） | 変更・実行のたびに確認 |
| **計画モード（Plan Mode）** | 読み取りのみ。まず計画を立て、実行はしない |
| auto-accept | 編集を自動承認（確認を省略） |

学習中は「通常モード」で、何が起きるか確認しながら進めるのがおすすめです。
計画モードは [docs/03](03-core-features.md) で扱います。

## セッションの継続

長い作業を中断して後で再開できます:

```bash
claude -c          # 直前の続きから
claude -r          # 一覧から選んで再開
```

## よく使う組み込みコマンド（抜粋）

`claude` 起動中に試してください:

```
/help        # コマンド一覧
/status      # 現在のモデル・設定・接続状況
/clear       # 会話履歴をクリア（コンテキストをリセット）
/agents      # 利用可能なサブエージェント一覧
/init        # CLAUDE.md を自動生成
```

### 試す: 状態確認

```
/status
```

使用中のモデルや権限モード、読み込まれている設定ファイルが確認できます。
